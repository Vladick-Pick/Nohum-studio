## Required Outputs

- shared Product Bet Card
- specialist-owned section sufficiency review
- ICP, buyer, user, payer, and excluded segments
- JTBD and workflow statement
- offer summary
- pricing summary
- organic/free traffic path
- selected test revision
- landing/waitlist validation surface
- observation window
- traffic source report
- financial model refs
- stack-fit verdict
- payment and measurement plan
- MVP in-scope list
- MVP out-of-scope list
- Gate B recommendation packet

## Rules

- optimize for first valid payment, not completeness
- keep exclusions explicit
- do not rewrite the frozen Idea Card after Gate A
- require specialists to write owned Product Bet Card sections, not loose notes
- if pricing is vague, Gate B is not ready
- if organic traffic path is vague, Gate B is not ready
- if selected test revision is missing, Gate B is not ready
- if landing/waitlist surface is missing, Gate B is not ready
- if observation window is not ready and no accepted-risk exception exists, Gate B is not ready
- if payment or measurement is vague, Gate B is not ready
- if the venture does not fit the canonical NoHum stack, Gate B is not ready without explicit board exception
- Gate B recommendation is not Gate B approval
