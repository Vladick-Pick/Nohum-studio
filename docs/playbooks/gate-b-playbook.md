# Gate B Playbook

Owners:
- `Launch Lead`
- `Product Launch Team`
- `CEO`

Expected outputs:
- Gate B packet
- PASS / FAIL / RETRY / ESCALATE verdict

Operating sequence:
1. Verify product definition, UX flow, pricing, measurement, and support-readiness prerequisites.
2. Verify the venture fits the canonical stack: `Next.js`, `Better Auth`, `PostgreSQL`, `Prisma`, `Railway`, `Lava.top`, `Plausible`, `Resend`, plus `pg-boss`, `R2`, and `Sentry` when needed.
3. Ensure Engineering, Marketing, and Support all have explicit downstream artifacts to consume.
4. Escalate unresolved payment, pricing, launch-readiness, or stack-fit ambiguity before build starts.
