# Venture Definition / Product Bet Validation Playbook

Owners:
- `Launch Lead`
- `Product Definer`
- `UX Researcher`
- `UX Architect`
- `UI Designer`
- `Pricing Strategist`

Expected outputs:
- Product Bet Validation packet
- launch-brief
- handoff dossier
- Gate B recommendation

Operating sequence:
1. Start from the Gate A approved `Idea Card` and Gate A decision, not from a blank page.
2. Lock ICP, JTBD, offer, pricing, UX flow, MVP boundary, validation risks, and test GTM into canonical artifacts.
3. Surface unresolved validation risks explicitly and route them before Gate B.
4. Do not hand work to Engineering until the definition packet and acceptance criteria are coherent.
5. Treat the Gate B recommendation as a review input, not build approval.
