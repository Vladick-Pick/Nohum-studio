# NoHum Studio Company Launch Sequence

## Goal

Turn the bootstrap package into a live Paperclip company with the smallest viable operating contour.

## Sequence

1. Import this package into Paperclip as a new company.
2. Verify that the imported company contains:
   - CEO
   - Research Lead
   - Launch Lead
3. Create the missing live entities inside the server company:
   - `Hypothesis Funnel`
   - `Studio OS`
   - `Bootstrap Company Access And Secrets` issue assigned to CEO
   - `Bootstrap Company` issue assigned to CEO
4. Let the CEO run the live `Bootstrap Company Access And Secrets` issue first.
5. Set company budget and agent budgets.
6. Create or verify the top-level company goal.
7. Enable the CEO heartbeat.
8. Let the CEO run the live `Bootstrap Company` issue.
9. Review and approve the CEO's initial strategy.
10. Verify that Research Lead and Launch Lead are active and correctly report to CEO.
11. Confirm that the first sourcing cycle starts in `Hypothesis Funnel`.
12. Move into normal operating cadence on the server runtime.

## Startup Shape

The intended day-1 company shape is:

- Board
  - CEO
    - Research Lead
    - Launch Lead

No full build or growth staff are required on day 1. Those roles can be hired later through the live company if the managers need more capacity.

## Day-1 Runtime Truth

After import, the live source of truth becomes the server Paperclip company runtime.

This repository remains useful for:

- preserving the original design
- iterating on future bootstrap versions
- recreating the company if needed

It is not the main live runtime control surface after import.
