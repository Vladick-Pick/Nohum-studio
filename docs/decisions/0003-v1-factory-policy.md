# Decision 0003: V1 Factory Policy

## Decision

NoHum Studio v1 operates under these hard constraints:

- one active venture plus one queued venture
- 14 days from `build_start_at` to valid first payment
- fast-cycle, self-serve, low-touch products only
- global / English-first
- first valid external payment is the pass signal
- TrustMRR-first sourcing bias

## Why

- launch capacity is scarce
- false positives are expensive
- budget discipline matters more than idea volume

## Implication

Any process or agent behavior that weakens these constraints is a policy violation, not a harmless optimization.
