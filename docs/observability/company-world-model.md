# Company World Model

The company world model is the machine-readable view of internal studio reality.

## It Must Track

- active and queued WIP
- blocked work
- stale work
- readiness debt
- runtime reliability
- skill and instruction drift
- budget posture
- capability gaps by team
- experiment backlog and current rollout ring

## Owners

- `Chief of Staff` owns prioritization and synthesis
- `Agent Mechanic` owns runtime and drift inputs
- team leads own their surface-level deltas

## Inputs

- factory health report
- org hygiene report
- runtime audit notes
- blocked-work packets
- reroute notes
- experiment records
- promotion decisions

## Rule

The company world model is derived from canonical artifacts. It is not a second source of truth.
