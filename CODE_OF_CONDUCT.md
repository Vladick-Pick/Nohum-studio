# Code of Conduct

NoHum is a rigorous public design project.

Treat other contributors accordingly.

## Expected Behavior

- challenge ideas directly, not people
- explain reasoning and tradeoffs
- cite runtime consequences when disagreeing
- be explicit about uncertainty
- keep discussion concrete and relevant to the module being changed

## Unacceptable Behavior

- harassment, insults, or personal attacks
- performative condescension
- hype-heavy noise that ignores the actual design problem
- derailing discussion away from the repo’s scope
- intentionally misrepresenting what the current system does or does not support

## Project Culture

The goal here is not to look visionary.

The goal is to design an organizational system that is:

- clearer
- more executable
- more auditable
- less person-dependent

That requires high standards and straightforward disagreement.

## Enforcement

The repository maintainer may remove comments, close discussions, or reject contributions that violate this code of conduct or repeatedly degrade the quality of collaboration.
