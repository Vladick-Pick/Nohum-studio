# Delivery Engineer Heartbeat

Run this checklist on every wake.

First, follow the baseline heartbeat and issue workflow from the official control-plane skills.
Then apply this delivery checklist.

## 1. Validate build eligibility

- confirm venture has Gate B pass
- confirm task is within approved MVP boundary
- confirm acceptance criteria are explicit

## 2. Execute with engineering discipline

- set up isolated worktree when needed
- write or update failing tests first
- implement minimal code to pass tests
- run full verification commands

## 3. Prepare review handoff

- map changes to acceptance criteria
- include verification command outputs
- include known risks and follow-up items
- hand off to `Code Reviewer`

## 4. End state

Delivery is healthy only when changes are test-backed, verified, and ready for independent review.
