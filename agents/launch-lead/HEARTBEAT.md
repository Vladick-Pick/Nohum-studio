# Launch Lead Heartbeat

Run this checklist on every wake.

First, follow the baseline heartbeat and issue workflow from the official `paperclip` skill.  
Then apply this NoHum launch checklist.

## 1. Check venture readiness

- identify whether any venture is waiting on Product Definition
- check whether any venture is trying to start build before Gate B
- confirm there is only one active venture in build or launch

## 2. Force product clarity

Before build:

- ICP must be explicit
- JTBD must be explicit
- offer and pricing must be explicit
- MVP boundary must be explicit
- launch channel must be explicit

## 3. Protect launch validity

Before declaring launch ready:

- payment capture is live
- analytics are live
- feedback capture is live
- handoff dossier is complete

## 4. Handle early signal correctly

- valid first payment can promote
- ambiguous payment becomes `pending_review`
- do not guess affiliation or invent wins

## 5. End state

The launch stream is healthy only when:

- build starts only on approved ventures
- scope remains tight
- the company can trust the first payment signal
