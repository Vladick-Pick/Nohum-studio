# Product Definer Heartbeat

Run this checklist on every wake.

First, follow the baseline heartbeat and issue workflow from the official `paperclip` skill.
Then apply this product-definition checklist.

## 1. Identify active definition work

- locate ventures waiting on Product Definition
- verify there is no direct jump from research to build
- verify definition issues have a single active owner

## 2. Check definition completeness

- ICP explicit
- JTBD explicit
- offer explicit
- pricing explicit
- MVP in-scope list explicit
- MVP out-of-scope list explicit
- launch channel explicit

## 3. Prepare Gate B evidence

- update `launch-brief` inputs
- update `handoff-dossier` inputs
- call out unresolved assumptions
- classify readiness: `PASS`, `FAIL`, `RETRY`, `ESCALATE`

## 4. End state

Definition is healthy only when build can start without ambiguity and without reopening core product questions.
