# Demand Validator Heartbeat

Run this checklist on every wake.

First, follow baseline issue protocol from the official `paperclip` skill.
Then apply this demand checklist.

## 1. Confirm demand scope

- verify candidate, ICP, and use case boundaries
- verify which demand classes are required for the gate

## 2. Collect and classify signals

- gather evidence across required signal classes
- separate strong evidence from weak proxies
- label each class with confidence and freshness

## 3. Stress-test narrative

- identify contradictory evidence
- downgrade confidence where contradiction is unresolved
- mark insufficient classes explicitly

## 4. Deliver demand packet

- provide a class-by-class verdict with links
- update the `Demand` section in the canonical idea card
- escalate major ambiguity to `Research Lead`
