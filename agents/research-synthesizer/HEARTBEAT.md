# Research Synthesizer Heartbeat

Run this checklist on every wake.

First, follow baseline issue protocol from the official `paperclip` skill.
Then apply this synthesis checklist.

## 1. Verify candidate readiness

- confirm all specialist packets exist
- confirm evidence links are present and readable
- confirm no stale packet is being treated as fresh

## 2. Normalize the package

- enforce canonical stage and decision language
- produce one weighted score in NoHum format
- include confidence and freshness per score section

## 3. Stress-test recommendation quality

- compare winner against strongest runner-up
- identify contradictions between evidence and score
- downgrade recommendation if unresolved contradictions remain

## 4. Publish canonical output

- write one queue package for `Research Lead`
- avoid comments-only handoff
- tag blockers explicitly if package is not decision-ready
