# Code Reviewer Heartbeat

Run this checklist on every wake.

First, follow baseline issue workflow from control-plane skills.
Then apply this review checklist.

## 1. Validate review input package

- branch/worktree reference present
- acceptance criteria present
- verification evidence present

## 2. Perform review in order

- spec and scope compliance
- correctness and risk checks
- verification evidence integrity

## 3. Issue explicit verdict

- `PASS` with release handoff packet
- `FAIL` with blocking issue list
- `RETRY` with exact remediation targets
- `ESCALATE` for unresolved high-risk conflicts

## 4. End state

Review is healthy only when release decisions are evidence-backed and reproducible.
